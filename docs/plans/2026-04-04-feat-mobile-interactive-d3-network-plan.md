# Mobile Interactive D3 Network Graph

Transform the Genre Map from a decorative display (opacity 0.4, non-interactive) into a fully interactive D3 visualization on mobile with touch gestures: pinch-zoom, tap-to-select, and drag-to-pan.

## Overview

The RA Genre Network graph currently uses force simulation to visualize musicological relationships between genres as a complex, organic node-link diagram. On mobile (≤768px), the graph is disabled (decorative only) to avoid clutter. This feature makes it fully interactive with touch-optimized gestures, enabling mobile users to explore the genre network intuitively.

**Key user benefit:** Mobile users can tap nodes to explore related genres, pinch to zoom into dense areas, and understand the "big picture" of how RA genres interconnect — mirroring the desktop experience.

## Problem Statement / Motivation

**Current state:** Mobile Genre Map tab shows a ghosted D3 graph (40% opacity, no touch interaction).

**User pain points:**
- Mobile users see the graph but can't interact with it
- No way to explore genre relationships on mobile
- Discrepancy between desktop (fully interactive) and mobile (decorative only) experiences

**Why now:** Mobile optimization is core to the project. Previous work (Phase 1–3) established bottom-sheet detail panels, 44px touch targets, and safe area support. D3 interactivity is the natural next step.

## Proposed Solution

**Touch-Optimized D3 Force Graph** (see brainstorm: `docs/brainstorms/2026-04-04-mobile-network-graph-brainstorm.md`):
- Fullscreen-ish viewport (primary focus)
- Respond to touch gestures: **pinch-zoom**, **tap-to-select**, drag-to-pan (optional)
- Reuse existing force simulation (no physics changes)
- Integrate with bottom-sheet detail panel (tap node → panel slides up)
- Maintain existing visual hierarchy and interactive states

#### Phase 2: Touch Gesture Handlers (Core + Polish)

**Goal:** Implement pinch-zoom, optional drag-pan, improved tap feedback.

**2a. Pinch-zoom Handler**

**Approach:** Extend existing D3 zoom behavior or add native touch handler.

D3 v7's `d3.zoom()` doesn't natively support iOS pinch-zoom in all browsers. Two options:

**Option A (Recommended):** Native `touchmove` listener with pinch detection
- Listen to `touchmove` events on SVG
- Detect 2-finger touch (e.touches.length === 2)
- Calculate distance between touches → scale delta
- Apply zoom via D3: `svg.transition().call(zoom.scaleTo, newScale)`
- Pros: Explicit control, works on all iOS/Android versions
- Cons: Requires custom gesture math

**Option B (Fallback):** Leverage browser defaults
- Test if D3 zoom already captures pinch on target devices
- If not working, fall back to Option A

**Implementation location:** After line 2378 (after `svg.call(zoom)`)

```javascript
// ── Mobile Touch Zoom (Pinch) ──────────────────────────────────────
let lastDistance = 0;

svg.on('touchmove', function(event) {
  if (event.touches.length !== 2) return;
  
  // Calculate distance between two fingers
  const dx = event.touches[0].clientX - event.touches[1].clientX;
  const dy = event.touches[0].clientY - event.touches[1].clientY;
  const distance = Math.sqrt(dx * dx + dy * dy);
  
  if (lastDistance === 0) {
    lastDistance = distance;
    return;
  }
  
  // Calculate scale delta
  const scale = distance / lastDistance;
  const newZoom = currentZoomK * scale;
  
  // Clamp to zoom bounds (0.2 to 5, per d3.zoom config)
  const clampedZoom = Math.max(0.2, Math.min(5, newZoom));
  
  // Apply zoom centered on midpoint of two fingers
  const midX = (event.touches[0].clientX + event.touches[1].clientX) / 2;
  const midY = (event.touches[0].clientY + event.touches[1].clientY) / 2;
  
  svg.call(zoom.scaleTo, clampedZoom);
  lastDistance = distance;
});

svg.on('touchend', () => {
  lastDistance = 0;
});
```

**2b. Drag-Pan Handler (Optional, Lower Priority)**

**Approach:** Add single-finger drag on SVG background to pan the camera (optional).

**Decision:** Test with users first. If users prefer only pinch-zoom (no pan), skip this.

**Implementation location:** After pinch-zoom handler

```javascript
// ── Mobile Touch Pan (Optional) ────────────────────────────────────
let panStartX = 0, panStartY = 0;
let isPanning = false;

svg.on('touchstart', function(event) {
  if (event.touches.length !== 1) return;  // only single finger
  isPanning = true;
  panStartX = event.touches[0].clientX;
  panStartY = event.touches[0].clientY;
});

svg.on('touchmove', function(event) {
  if (!isPanning || event.touches.length !== 1) return;
  
  const dx = event.touches[0].clientX - panStartX;
  const dy = event.touches[0].clientY - panStartY;
  
  // Apply translation to graph
  const currentTransform = d3.zoomTransform(svg.node());
  const newTransform = currentTransform.translate(dx / currentTransform.k, dy / currentTransform.k);
  svg.call(zoom.transform, newTransform);
  
  panStartX = event.touches[0].clientX;
  panStartY = event.touches[0].clientY;
});

svg.on('touchend', () => {
  isPanning = false;
});
```

**2c. Tap-to-Select Enhancement**

**Current behavior:** Click handler fires on touch (line 2573), opens detail panel via `selectGenreNode()`.

**Enhancement:** Add visual feedback for tap (ripple effect or highlight) + longer tap duration threshold (optional).

**Implementation location:** Modify click handler at line 2573

```javascript
nodeG.on('click', (event, d) => {
  // Existing logic
  selectGenreNode(d.id);
  
  // Optional: Add ripple effect (CSS animation)
  const ripple = d3.select(event.currentTarget).append('circle')
    .attr('cx', d => nodeRadius(d.count))
    .attr('cy', d => nodeRadius(d.count))
    .attr('r', 0)
    .attr('fill', 'rgba(255, 255, 255, 0.5)')
    .attr('class', 'tap-ripple');
  
  ripple.transition()
    .duration(300)
    .attr('r', d => nodeRadius(d.count) * 2)
    .style('opacity', 0)
    .remove();
});
```

### System-Wide Impact

#### Interaction Graph (Touch → Detail Panel)
```
User taps node on D3 graph
  ↓
D3 click handler fires (line 2573)
  ↓
selectGenreNode(d.id) called
  ↓
- Node highlighted (glow intensity increases, neighbors highlighted)
- Detail panel class updated: detailPanel.classList.remove('translate-full-down')
- Detail panel slides up (CSS translateY animation, 300ms)
  ↓
Detail panel visible (50vh), shows genre details + LLM labels + RA Tags
  ↓
User can:
  - Swipe detail panel up → expand to 90vh
  - Click "Listen" → opens SoundCloud
  - Click genre chips → highlight in graph + optional: zoom to node
  - Swipe down → close detail panel
```

#### Error Propagation
- **Touch handler errors:** Will not break entire page (listeners are isolated)
- **D3 zoom errors:** Already tested (existing code), catch via `try-catch` if adding new math
- **Detail panel interaction:** Already has error handling for missing elements (checks `if (detailPanel)` before accessing)

#### State Lifecycle Risks
- **D3 state:** `currentZoomK` and `highlightedNodes` stored in module scope. Touch handlers read/modify these.
- **Risk:** Multiple gesture handlers firing simultaneously (e.g., pinch + pan). Mitigation: Each handler checks `lastDistance === 0` to avoid conflicting updates.
- **DOM state:** Detail panel uses CSS class toggle (`.translate-full-down`). Touch handlers will modify this safely (class already exists in codebase).

#### API Surface Parity
- **Desktop:** Hover → tooltip + highlight neighbors; click → detail panel
- **Mobile:** Tap → detail panel (no hover); pinch-zoom → same as desktop scroll-wheel zoom; drag-pan → same as desktop pan
- **Consistency:** Mobile gestures map 1:1 to desktop actions. No new behaviors.

#### Integration Test Scenarios
1. **Pinch-zoom + node click**: User pinches to zoom in, taps a small node, detail panel opens with correct genre
2. **Drag-pan + detail panel**: User pans graph, then taps node in new position, panel opens correctly
3. **Rapid taps**: User taps 3 nodes in succession → each opens detail panel, previous selections clear (test race conditions)
4. **Zoom boundary**: User pinches beyond 5x zoom limit → graph stops zooming, doesn't break simulation
5. **Touch during simulation**: User taps while force simulation is running → selection works, simulation doesn't stutter

## Risk Analysis & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|----|
| **Pinch-zoom not working on iOS** | Users can't zoom on primary platform | Medium | Test on actual iOS device early (Phase 2 beginning); implement D3 fallback if native doesn't work |
| **Touch events conflict with D3 drag** | Node dragging breaks or behaves unexpectedly | Low | D3 drag uses `d3.drag()` which handles touch internally; test all 3 gestures together |
| **JavaScript redeclaration error** | Entire page stops loading | Low | Search file for existing `const/let` declarations before adding code; reuse existing refs (enforce pattern review) |
| **Performance: Battery drain on older devices** | Mobile users abandon app due to battery impact | Medium | Monitor frame rate + CPU during testing; defer optimization to Phase 3 if needed |
| **Detail panel overlaps with graph gestures** | Users can't tap nodes when panel is open | Low | Z-index ordering already established (detail panel above graph); verify in testing |
| **Search box placement conflicts with graph** | Graph at small zoom overlapped by search UI | Low | Keep search hidden on mobile (Phase 1); reconsider placement in Phase 3 polish |
